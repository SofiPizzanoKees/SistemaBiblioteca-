# SistemaBiblioteca
Este proyecto (FINAL AEDII) tiene como objetivo gestionar el préstamo de libros, el registro y administración de usuarios, el control de devoluciones, la disponibilidad de los libros, y la reserva de libros a través de la web para un instituto.
Objetivos del Sistema


## Fase 1 (MVP - Mínimo Producto Viable)
📖 Gestión completa de libros, autores y editoriales.

👤 Registro y administración de usuarios.

🔄 Control de préstamos y devoluciones.

🔍 Consulta de disponibilidad de libros en tiempo real.

📊 Generación de estadísticas de préstamos y reportes de libros más solicitados.

## Fase 2 (Próximas funcionalidades)
📅 Reserva de libros en línea.

🔔 Notificaciones automáticas (vencimientos, disponibilidad, etc.).

📈 Estadísticas avanzadas de uso del material bibliográfico.


CRUD (Alta, Baja Lógica, Modificación) de:
- Libros.
- Usuarios.

Gestión de préstamos:
Registrar préstamos y devoluciones.

Consultas avanzadas:
- Libros prestados.
- Libros disponibles.
- Préstamos vencidos.

👤 Usuario
- Inicio de sesión.
- Búsqueda y filtrado de libros (mínimo 2 filtros).
- Consulta de disponibilidad en tiempo real.
- Historial de préstamos propio.
- Solicitud de préstamos.


## 🛠️ Características Técnicas
✅ Mínimo 2 filtros para búsquedas avanzadas (ej. por autor, editorial, año, género, etc.).

✅ Mínimo 2 listados dinámicos (ej. libros más prestados, usuarios con más préstamos, préstamos vencidos, etc.).

✅ Arquitectura escalable preparada para futuras integraciones.

✅ Persistencia de datos robusta y segura.

✅ Interfaz intuitiva adaptada a cada perfil de usuario.
